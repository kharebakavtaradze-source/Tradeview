ENGINE: SQLAlchemy async ORM
PRIMARY DB: PostgreSQL (asyncpg driver) via Railway plugin
FALLBACK: SQLite (aiosqlite) — local dev only — DATA LOST ON RESTART

DB URL detection (database.py):
  DATABASE_URL env var → use PostgreSQL
  Not set             → SQLite fallback + WARNING logged at startup
  "postgresql://"     → auto-converted to "postgresql+asyncpg://"

IMPORTANT: Never set DATABASE_URL = "" in railway.toml — this overrides the
Railway plugin injection and causes SQLite fallback on every deploy.

Startup: init_db() → create_all tables + run_migrations (safe ALTER TABLE)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Tracks consecutive scan appearances for ARM+ tickers.

  id             INTEGER PK
  symbol         VARCHAR(10) UNIQUE  (indexed)
  streak_start   DATE NOT NULL
  streak_days    INTEGER default=1
  avg_score      FLOAT default=0
  avg_cmf_pctl   FLOAT default=0
  avg_vol_ratio  FLOAT default=0
  avg_hype       INTEGER default=0
  tier           VARCHAR(20) nullable   (best tier seen in streak)
  wyckoff        VARCHAR(30) nullable   (Wyckoff state)
  last_seen      DATE NOT NULL
  alerted        INTEGER default=0   (bitmask: bit0=day3 sent, bit1=day5 sent)

Notes:
  - One row per symbol (upserted each scan via update_pattern_streaks)
  - Streak resets if symbol disappears for 2+ days (last_seen gap > 1 day)
  - Rolling averages: avg = (prev_avg * (days-1) + today) / days
  - Telegram alerts sent at day 3 (bit0) and day 5 (bit1) milestones
  - Once bit is set, alert not re-sent (prevents spam)
