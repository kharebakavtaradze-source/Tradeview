GET /api/scan/latest
  Returns most recent scan results.
  Strips "candles" field from results list (use /ticker/{symbol} for candles).
  Response: {results[], scanned_at, total, tier_counts, sector_strength,
             market_regime, symbol_sources, ...}

GET /api/scan/history?days=30
  Returns scan summary for last N days (max 90).
  Response: {history[], days}

GET /api/scan/run  (also POST /api/scan/run)
  Manually triggers a scan in background.
  Returns immediately: {status, message}
  Prevents double runs via _scan_running flag.

GET /api/ticker/{symbol}
  Returns full detail for one ticker from latest scan, including candles (100 bars).
  If not in scan: fetches live data from Yahoo Finance on-demand.
  Returns: full ticker dict with candles[], source ("scan" or "live")

  Returns performance stats (win rate, avg gain/loss, by tier, by Wyckoff, etc.)

GET  /api/journal/suggest-levels?symbol=AAPL&entry=15.50&tier=ARM
  Returns ATR-based suggested stop and target.
  Stop multipliers by tier: FIRE=1.2x, ARM=1.5x, BASE=2.0x, STEALTH=1.5x
  Target = stop_distance * 2.5 (guarantees 2.5:1 R/R)
  Falls back to 3% of entry if ATR not available.
  Response: {symbol, entry, stop, target, stop_pct, target_pct, rr_ratio,
             atr_used, stop_mult, target_mult}

GET  /api/journal/adaptive-weights
  Returns current adaptive scoring weights derived from closed trades.
  Response: {cmf_weight, wyckoff_weight, vol_weight, hype_weight, count, ...}

GET  /api/journal/{id}
  Returns single journal entry by ID.

POST /api/journal
  Create new journal entry.
  Body: {symbol, entry_price, entry_date, tier, stop_loss?, target_price?,
         notes?, entry_wyckoff?, entry_cmf_pctl?, entry_vol_ratio?,
         entry_hype?, catalyst?, tags?}
  Validates R/R ratio if stop_loss + target_price provided:
    BLOCK if stop >= entry or target <= entry
    BLOCK if R/R < 1.0
    WARN  if R/R < 2.0 (still saved, warning returned)

PUT  /api/journal/{id}
  Update journal entry (any field).
  Recalculates gain_pct on save.

DELETE /api/journal/{id}
  Delete journal entry.
